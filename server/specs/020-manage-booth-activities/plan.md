# Implementation Plan: Manage Booth Activities

**Branch**: `020-manage-booth-activities` | **Date**: 2026-03-21 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/020-manage-booth-activities/spec.md`

## Summary

Add CRUD endpoints (`GET`, `POST`, `PUT`, `DELETE`) for booth activities scoped to a single partnership, at path `/events/{eventSlug}/partnerships/{partnershipId}/activities`. Routes are fully public (no auth). Access is gated by a per-request booth-eligibility check (`sponsoring_options.selectable_descriptor = BOOTH` via `PartnershipOptionsTable` join). The `WebhookPartnershipPlugin` is installed on the route group to fire webhooks automatically after each successful mutation.

## Key Design Decisions

See [research.md](research.md) for full rationale. Summary:

1. **Module**: Inside existing `partnership` module (not a new module)
2. **Path param**: `{partnershipId}` UUID — required by `WebhookPartnershipPlugin`
3. **Booth check**: `PartnershipOptionsTable` ⋈ `SponsoringOptionsTable` where `selectableDescriptor = BOOTH` → `403` if false
4. **Webhook**: `install(WebhookPartnershipPlugin)` on route group — no manual calls
5. **Sorting**: `startTime ASC NULLS LAST, createdAt ASC`
6. **Auth**: None — fully public routes

## API Endpoints

| Method | Path | Success | Errors |
|--------|------|---------|--------|
| `GET` | `/events/{eventSlug}/partnerships/{partnershipId}/activities` | 200 | 403, 404 |
| `POST` | `/events/{eventSlug}/partnerships/{partnershipId}/activities` | 201 | 400, 403, 404 |
| `PUT` | `/events/{eventSlug}/partnerships/{partnershipId}/activities/{activityId}` | 200 | 400, 403, 404 |
| `DELETE` | `/events/{eventSlug}/partnerships/{partnershipId}/activities/{activityId}` | 204 | 403, 404 |

Full contract: [contracts/openapi.yaml](contracts/openapi.yaml)

## Data Flow

```
Client → BoothActivityRoutes
           ↓ check partnership exists       → 404 if not
           ↓ check booth eligibility        → 403 if no BOOTH option
           ↓ validate request body (schema) → 400 if invalid
           ↓ validate startTime < endTime   → 400 if violated
           ↓ BoothActivityRepository (data operation)
           ↓ respond (201/200/204)
           ↓ WebhookPartnershipPlugin fires (onCallRespond hook)
```

## Implementation Notes

- **Exception→HTTP mapping**: `ForbiddenException` → 403, `NotFoundException` → 404 (via existing `StatusPages` in `App.kt`); schema validation → 400
- **activityId scope check**: on PUT/DELETE, verify `activity.partnershipId == partnershipId` from path before operating; return 404 if mismatch
- **`BoothActivityModule.kt`**: register `BoothActivityRepositoryExposed` as `BoothActivityRepository` singleton in Koin; include this module in `App.kt`
- **MigrationRegistry**: register `BoothActivitiesTable` so SchemaUtils creates the table on startup
